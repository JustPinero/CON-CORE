## Objective

Implement Claude API sender categorization. For a selected sender, sample N messages and send to Claude API for category breakdown (Promotions/Transactional/Newsletter/Personal/Work/System percentages). Server-side proxy route. Returns JSON category breakdown. Generate a one-line dossier summary per sender.

## Acceptance Criteria

- [ ] POST /api/claude/analyze-sender accepts sender_address, returns category_breakdown + dossier
- [ ] Samples up to 20 recent messages per sender for analysis
- [ ] Claude prompt returns valid JSON: {promo: %, transactional: %, work: %, personal: %, newsletter: %, system: %}
- [ ] Dossier is a one-line summary (e.g., "RETAIL ELECTRONICS PROMOTIONS — WEEKLY DEALS")
- [ ] Response JSON parsing strips markdown code fences
- [ ] AbortController timeout (15s) on Claude API call
- [ ] Client-side claude.ts service wraps the call

## Files to Touch

api/claude/analyze-sender.ts, src/services/claude.ts

## Tests to Write

- /api/claude/analyze-sender returns expected shape
- Category percentages sum to ~100
- Markdown code fence stripping works
- AbortController timeout triggers after 15s
- Invalid Claude response handled gracefully
