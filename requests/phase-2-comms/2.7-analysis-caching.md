## Objective

Cache Claude API analysis results in Supabase. Before calling Claude API for a sender, check if cached analysis exists and is fresh (less than 7 days old). If cached, return cached result. If stale or missing, call Claude API and cache the result. Update email_sources table with analysis data.

## Acceptance Criteria

- [ ] Analysis check queries Supabase before calling Claude API
- [ ] Fresh cache (< 7 days) returns immediately without API call
- [ ] Stale or missing cache triggers Claude API call
- [ ] Successful analysis cached to email_sources table
- [ ] Cache includes: category_breakdown, dossier_summary, last_analyzed timestamp
- [ ] Force-refresh option bypasses cache
- [ ] Cache utility is reusable for other Claude API responses

## Files to Touch

src/utils/cache.ts, api/claude/analyze-sender.ts (add cache layer), src/services/gmail.ts (check cache first)

## Tests to Write

- Cache returns stored result for fresh analysis
- Cache triggers API call for stale analysis (> 7 days)
- Cache triggers API call for missing analysis
- Force-refresh bypasses cache
- Cache stores result after successful API call
- Cache utility works for different data types

## Dependencies

2.3 (Claude analysis), 1.7 (Supabase client)
