## Objective

Register COMMS terminal commands: `comms list` (show senders), `comms analyze <sender>` (trigger analysis), `comms delete <sender> --category <cat>` (batch delete), `comms archive <sender> --category <cat>` (batch archive), `comms unsubscribe <sender>`. All commands use the same underlying service functions as the GUI.

## Acceptance Criteria

- [ ] `comms list` outputs sender list with counts
- [ ] `comms analyze <sender>` outputs category breakdown
- [ ] `comms delete <sender> --category promo` executes batch delete
- [ ] `comms archive <sender> --category promo` executes batch archive
- [ ] `comms unsubscribe <sender>` executes unsubscribe
- [ ] All commands registered in CommandRegistry with help text
- [ ] Tab completion works for `comms` subcommands
- [ ] Commands share service functions with GUI (no duplication)

## Files to Touch

src/stations/comms/comms.commands.ts, src/stations/terminal/CommandRegistry.ts

## Tests to Write

- `comms list` calls gmail.getSenders()
- `comms analyze` calls claude.analyzeSender()
- `comms delete` calls gmail.batchDelete() with correct params
- `comms archive` calls gmail.batchArchive() with correct params
- `comms unsubscribe` calls gmail.unsubscribe()
- All commands registered in CommandRegistry
- Tab completion returns comms subcommands

## Dependencies

2.1-2.7 (all COMMS features), 1.8-1.9 (Terminal)
