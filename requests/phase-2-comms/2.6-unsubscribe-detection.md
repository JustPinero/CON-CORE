## Objective

Detect unsubscribe vectors for each sender: check for List-Unsubscribe header in emails. If present, extract unsubscribe URL or mailto. Add UNSUBSCRIBE button to action buttons when vector exists. Execute unsubscribe via extracted method.

## Acceptance Criteria

- [ ] POST /api/gmail/unsubscribe checks List-Unsubscribe header
- [ ] Extracts HTTP URL or mailto unsubscribe vector
- [ ] UNSUBSCRIBE button appears only when vector exists
- [ ] Executing unsubscribe follows the extracted URL/mailto
- [ ] unsubscribe_vector boolean stored on EmailSource record
- [ ] Handles missing/malformed List-Unsubscribe headers gracefully

## Files to Touch

api/gmail/unsubscribe.ts, src/stations/comms/SourceIntel.tsx (add unsubscribe button)

## Tests to Write

- Unsubscribe route extracts HTTP URL from header
- Unsubscribe route extracts mailto from header
- Unsubscribe route handles missing header
- UNSUBSCRIBE button only renders when vector exists
- Unsubscribe execution follows correct method

## Dependencies

2.1, 2.5 (sender data, action buttons)
