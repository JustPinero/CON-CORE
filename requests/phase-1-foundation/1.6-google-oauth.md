## Objective

Implement Google OAuth2 flow: login API route initiates OAuth redirect, callback route exchanges code for tokens, refresh route handles token refresh. Store encrypted tokens in Supabase. Create useAuth hook for frontend auth state. Login button on homepage if not authenticated.

## Acceptance Criteria

- [ ] GET /api/auth/login redirects to Google OAuth consent screen
- [ ] GET /api/auth/callback exchanges code for tokens, stores encrypted in Supabase
- [ ] POST /api/auth/refresh refreshes expired access token
- [ ] Tokens encrypted with SESSION_SECRET before Supabase storage
- [ ] useAuth hook provides: isAuthenticated, login(), logout(), loading state
- [ ] Homepage shows LOGIN button when not authenticated
- [ ] Scopes include: Gmail read/modify, Calendar read/write
- [ ] No tokens or secrets in client-side code

## Files to Touch

api/auth/login.ts, api/auth/callback.ts, api/auth/refresh.ts, src/hooks/useAuth.ts, src/services/auth.ts, src/components/Homepage.tsx (login button)

## Tests to Write

- /api/auth/login returns redirect URL with correct scopes
- /api/auth/callback stores encrypted tokens
- /api/auth/refresh returns new access token
- useAuth returns loading state initially
- useAuth returns authenticated state after login
- No secrets exposed in client bundle (grep test)
