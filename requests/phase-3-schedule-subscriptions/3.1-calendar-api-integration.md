## Objective

Implement Google Calendar API integration: read events in a date range, create single events, list user's calendars. Server-side routes with auth token forwarding.

## Acceptance Criteria

- [ ] GET /api/calendar/events returns events for date range
- [ ] POST /api/calendar/batch-create creates multiple events
- [ ] Handles multiple calendars (user selects which calendar)
- [ ] Respects Calendar API rate limits
- [ ] Client-side calendar.ts service wraps API calls
- [ ] Error handling for expired tokens

## Files to Touch

api/calendar/events.ts, api/calendar/batch-create.ts, src/services/calendar.ts

## Tests to Write

- /api/calendar/events returns events for date range
- /api/calendar/batch-create creates events
- Client service handles expired token refresh
- Rate limiting on Calendar API calls

## Dependencies

1.6, 1.7 (auth + API structure)
