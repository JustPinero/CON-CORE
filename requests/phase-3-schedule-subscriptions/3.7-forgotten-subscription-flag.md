## Objective

Flag "forgotten" subscriptions: services where receipt emails exist but there are no recent usage signals (no non-receipt emails, no calendar events referencing the service). Mark as usage_status: "forgotten" in Supabase.

## Acceptance Criteria

- [ ] Scan for usage signals beyond receipt emails
- [ ] Flag subscriptions with no usage signals as "forgotten"
- [ ] "FORGOTTEN" badge on subscription list entries
- [ ] Forgotten subscriptions highlighted with warning color
- [ ] Usage status stored in Supabase (active vs forgotten)

## Files to Touch

api/claude/detect-subscriptions.ts (add usage signal check), src/stations/subscriptions/SubscrList.tsx (forgotten badge)

## Tests to Write

- Forgotten flag set for receipt-only subscriptions
- Active flag set for subscriptions with usage signals
- FORGOTTEN badge renders with warning color
- Usage status persists to Supabase

## Dependencies

3.5, 3.6 (subscription detection + UI)
