## Objective

Implement conflict detection: before deploying a schedule, check for overlapping events in the target date range. Show conflicts with details. Allow user to proceed or cancel.

## Acceptance Criteria

- [ ] POST /api/calendar/conflicts checks for overlaps in date range
- [ ] Returns list of conflicts: new event vs existing event with times
- [ ] Conflict UI shows side-by-side comparison
- [ ] User can proceed (deploy anyway) or cancel
- [ ] Works across multiple calendars

## Files to Touch

api/calendar/conflicts.ts, src/stations/schedule/DeploySchedule.tsx (conflict UI)

## Tests to Write

- Conflict detection finds overlapping events
- No false positives for adjacent (non-overlapping) events
- Handles events across multiple calendars
- Conflict UI renders conflict details

## Dependencies

3.1, 3.3 (Calendar API, deploy action)
