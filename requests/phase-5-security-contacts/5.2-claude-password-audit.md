## Objective

Send password data to Claude API for security analysis. Detect: reused passwords, weak passwords, forgotten accounts. Score each account GREEN/YELLOW/RED. Response held in state only — Claude API response not cached (security data).

## Acceptance Criteria

- [ ] POST /api/claude/audit-passwords accepts password records
- [ ] Claude identifies reused passwords (same password across accounts)
- [ ] Claude identifies weak passwords (short, common, no complexity)
- [ ] Claude flags forgotten accounts (old, unused services)
- [ ] Each account scored: GREEN (safe), YELLOW (concern), RED (critical)
- [ ] Issues array per account with specific problems
- [ ] Recommendation text per account
- [ ] Response NOT cached in Supabase (security data)
- [ ] AbortController timeout

## Files to Touch

api/claude/audit-passwords.ts, src/services/claude.ts

## Tests to Write

- Reused passwords detected
- Weak passwords scored RED
- Strong unique passwords scored GREEN
- Response shape matches PasswordAuditResult type
- No caching (verify no Supabase writes)
- AbortController timeout

## Dependencies

5.1, 1.7 (password data, API structure)
